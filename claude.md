项目背景

项目名称：“深海幽灵”——基于端边云协同与多级算力路由的潜艇集群调度系统
硬件背景设定：

边缘计算节点 (Edge Node)： 部署于水面母舰，配置相当于 10 张 H100 显卡算力，负责区域性 L1 战术处理。

云计算中心 (Cloud Center)： 部署于陆地数据中心，超算级别算力，负责 L2 战略统筹与边缘节点溢出接管。

核心卖点：

多级算力路由与动态溢出 (Spillover)： 结合规则引擎 (L0)、神经网络 (L1/L2)，并引入基于硬件负载的动态任务上抛机制。

双缓冲机制 (Ping-Pong DB)： 解决边缘网关层高并发写与 L1 层快 AI 慢读取的锁竞争冲突。

多视角指挥舱： 支持路由切换，提供中心全局视角 (/center) 与边缘节点监控视角 (/node)。

核心开发原则（非常重要）

该Mock的Mock： 底层通信、PID控制请使用优雅的代码模拟（Mock）（如 asyncio.sleep）。

多级评估路由器必须真实运作： 系统启动时，必须使用 Python（纯 numpy 或 scikit-learn）自动生成一批历史数据，现场训练一个简易分类器（注意使用GPU），将其用于数据流分发判断。

双缓冲必须真实体现： 必须用代码真实体现 DB_A 和 DB_B 的积攒、指针的翻转、读写分离的互斥。

去 AI 化的极客 UI： 前端必须使用 React Router 实现多页面。UI 采用硬核的工业级/军工级大屏控制台风格。深色模式，大字号数据，等宽字体。绝对不要出现闪烁星星、机器人等幼稚图标。

核心架构与功能需求分解

1. 核心心脏：三级任务评估优化器 (Hybrid Evaluator)

网关收到水下潜艇集群的高频数据后，进入路由分发环节：

Step 1: 规则引擎前置拦截 (L0 - 纯粹的生死攸关)

逻辑： 纯硬代码 if-else。如发现紧急标识，完全绕过神经网络，极速本地处理并在终端拉响警报。

Step 2: 神经网络智能分流 (L1 vs L2)

逻辑： 排除 L0 的数据，提取特征，输入启动时训练好的神经网络模型中。

Step 3: 动态算力溢出监控 (The Spillover)

如果模型判定为 L1 (边缘节点处理)，系统会检查当前 /node 边缘节点的 GPU 模拟负载率。

正常情况 (<85% 负载)： 导入边缘节点的双缓冲池 (Ping-Pong DB)，由本地 10x H100 消化。

爆满情况 (>85% 负载)： 触发算力溢出 (Compute Spillover)，将本应属于 L1 的任务强行标记为 [L1-ESCALATED]，并上抛至数据中心的 L2 处理队列，保护边缘节点不被压垮。

L2 (战略任务)： 无论是因为模型判定还是算力溢出，统一压入“云端上传队列”，交由超算中心处理。

2. L1 层的“双缓冲”机制（Double-Buffering）

针对进入边缘节点 L1 的任务：

维护内存字典 DB_A 和 DB_B，及指针 active_write_db = 'A'。

定时翻转指针。后台协程遍历只读旧 DB 模拟轻量级模型分析，产出战术指令。

3. 前端多屏可视化 (React + Tailwind + React Router)

界面必须提供前端路由切换机制，包含两个核心视图：

视图一：边缘节点监控控制台 (/node)

视觉焦点： 硬件压榨与双缓冲流转。

左侧：硬件遥测 (Hardware Telemetry)。 绘制 10 个代表 H100 GPU 的进度条/方块，模拟展示实时负载（Load: 0%-100%）、VRAM 占用。

中侧：双缓冲池微观视图 (Buffer Core)。 动态展示 DB_A 和 DB_B 的 [写入中] 与 [处理中] 翻转动画。

右侧：溢出告警日志。 重点展示任务吞吐量，以及当负载超过 85% 时，触发的红色警告：[WARNING] GPU Load Critical. Escaping tasks to Cloud Center.

视图二：超算中心全局大屏 (/center)

视觉焦点： 全局算力路由与集群阵列。

左侧：潜艇集群全局阵列 (Swarm Map)。 网格展示潜艇状态，L0 报警时红框闪烁。

中侧：三级路由大脑 (Global Router)。 宏观展示数据流拆分流向：[L0 本地熔断], [L1 边缘计算], [L2 云端超算 (含边缘溢出)]。

右侧：超算深加工日志。 模拟耗时较长、参数极大的深度模型评估结果打印。
